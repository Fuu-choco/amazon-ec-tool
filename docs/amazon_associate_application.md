# Amazonアソシエイトプログラム申請ガイド

## 申請時の情報入力

### 1. ウェブサイト情報の入力

#### 1.1 必須項目
- **サイト名**: `Amazon商品データ管理システム`
- **サイトURL**: `http://localhost:8000` (開発環境)
- **サイト説明**: `Amazon商品データを取得・分析し、Googleスプレッドシートに反映するシステム`

#### 1.2 推奨入力内容
```
サイト名: Amazon商品データ管理システム
サイトURL: http://localhost:8000
サイト説明: Amazon商品の価格・在庫情報を自動取得し、Googleスプレッドシートで管理するシステムです。商品分析、価格追跡、在庫監視機能を提供します。
```

### 2. モバイルアプリ情報の入力

#### 2.1 開発中のアプリの場合
- **アプリ名**: `Amazon商品管理アプリ`
- **プラットフォーム**: `Webアプリケーション`
- **アプリURL**: `http://localhost:8000`
- **アプリ説明**: `Amazon商品データ管理のWebアプリケーション`

#### 2.2 推奨入力内容
```
アプリ名: Amazon商品管理アプリ
プラットフォーム: Webアプリケーション
アプリURL: http://localhost:8000
アプリ説明: Amazon商品の価格・在庫情報を管理するWebアプリケーションです。商品検索、価格追跡、データ分析機能を提供します。
```

### 3. 申請時の注意点

#### 3.1 開発環境での申請
- **開発中**であることを明記
- **localhost**での動作を説明
- **将来的な本番環境**への移行予定を記載

#### 3.2 申請理由の記載
```
申請理由: 
Amazon商品データを自動取得し、Googleスプレッドシートで管理するシステムを開発しています。
商品の価格変動分析、在庫監視、競合分析などの機能を提供する予定です。
```

### 4. 申請手順の詳細

#### 4.1 申請ページでの入力例
```
1. アカウント情報
   - メールアドレス: [あなたのメールアドレス]
   - パスワード: [安全なパスワード]

2. サイト情報
   - サイト名: Amazon商品データ管理システム
   - サイトURL: http://localhost:8000
   - サイト説明: Amazon商品データの自動取得・分析システム

3. アプリ情報
   - アプリ名: Amazon商品管理アプリ
   - プラットフォーム: Webアプリケーション
   - アプリURL: http://localhost:8000
   - アプリ説明: 商品データ管理Webアプリケーション

4. 申請理由
   - 開発中のシステムで、Amazon商品データを分析・管理するため
   - 価格追跡、在庫監視、競合分析機能を提供予定
```

### 5. 審査期間中の対応

#### 5.1 審査期間
- **通常1-2週間**で審査結果が通知
- **承認されるまで**実際のAPI呼び出しは不可

#### 5.2 審査期間中の開発
- **モックデータ**を使用して開発継続
- **テストコード**の充実
- **Google Sheets連携**の実装

### 6. 承認後の設定

#### 6.1 取得する情報
- **Associate Tag**: `yourname-20` のような形式
- **トラッキングID**: 商品リンクに含めるID

#### 6.2 設定ファイルの更新
```bash
# .envファイルを更新
ASSOCIATE_TAG=yourname-20
```

### 7. トラブルシューティング

#### 7.1 申請が拒否された場合
- **サイト内容**をより詳細に説明
- **開発目的**を明確に記載
- **将来的な計画**を追加

#### 7.2 審査が遅い場合
- **1週間後**に問い合わせ可能
- **申請内容**の見直しを検討

### 8. 開発中の代替手段

#### 8.1 モックデータでの開発
```python
# モックデータを使用して開発継続
mock_associate_tag = "test-associate-20"
```

#### 8.2 テスト環境での動作確認
```python
# テスト用の設定
TEST_MODE = True
MOCK_API_RESPONSES = True
```
